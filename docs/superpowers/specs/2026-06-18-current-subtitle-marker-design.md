# いま再生中の字幕マーカー設計

**ステータス**: 確定

**作成日**: 2026-06-18

## 背景

画面右側の字幕履歴パネル（`transcriptPanel.ts`）には直近の字幕が時系列で並ぶが、
動画のいまの再生位置がどの行に対応するのかが分からない。再生に合わせて
「いま喋っている字幕」がひと目で分かるマーカーを追加し、学習中の現在位置の把握と、
過去行クリックでの巻き戻しをよりスムーズにする。

## 要望

字幕履歴パネルに「現在再生されている箇所の該当字幕がどこなのかがひと目で分かるマーク」
を追加する。

## 既存実装の前提

- 各履歴エントリ `TranscriptEntry` は `videoTime`（その字幕が出た瞬間の動画再生位置・秒）を持つ。
- 動画の現在位置は `findVideo()?.currentTime` で取得できる（`videoControl.ts`）。
- パネルは `append(entry)` で行を時系列に追加し、行クリックで `onSeek(entry.videoTime)` を呼ぶ。
- 履歴は再生位置フロンティア方式で記録されるため、各行の `videoTime` はおおむね昇順。
- メディアイベント（`loadstart` 等）は `document` のキャプチャ段で購読する既存パターンがある（`index.ts`）。

## 設計

### 現在行の判定ロジック

- 「`videoTime <= currentTime`（+小さな許容誤差）を満たす行のうち `videoTime` が最大の行」を
  **いま再生中の行**とする。
- 境界ケース:
  - `currentTime` が最初の行の `videoTime` より前 → どの行もアクティブにしない（マークなし）。
  - 履歴が空 → 何もしない。
  - 巻き戻して履歴の途中を再生 → その位置に対応する行がアクティブになる。
- 判定はパネルが内部に持つ行データに対して行う（パネル自己完結）。
  エントリ数は1エピソードで高々数百行のため、線形走査で十分。

### 再生位置の監視（配線）

- `index.ts` で動画の再生位置を監視し、変化のたびにパネルへ通知する。
- 監視のきっかけ:
  - `timeupdate`（再生中に約0.25秒ごと）
  - `seeked`（履歴クリックや再生バー操作でのシーク直後）
  - 新しい字幕を `append` した直後
- 動画要素が広告等で入れ替わっても拾えるよう、`document` のキャプチャ段で
  `timeupdate` / `seeked` を購読する（既存の `loadstart` 購読と同じ手口）。

### パネル API の追加

- `transcriptPanel.ts` に `updateActiveByTime(currentTime: number): void` を追加。
  - 内部で現在行を判定し、その行に `.active` クラスを付け、他行から外す。
  - アクティブ行が変わったときだけ「賢い追従スクロール」を行う。
- 行と `videoTime` の対応は `append` 時に保持する（id 順に並ぶ内部配列、または各行へ `videoTime` を保持）。

### 見た目（`.row.active`）

- 行の左端に青い縦バー（4px 程度）＋淡い青の背景。
- 既存のホバー色 `rgba(86, 156, 255, 0.18)` と色味を揃え、馴染ませる。
- マーカーは控えめにし、字幕本文・訳の可読性を損なわない。

### 賢い追従スクロール

- アクティブ行が変わったら、その行が見える位置へスクロールする。
- ただしユーザーが手動でスクロールしている間は追従を一時停止する
  （過去を遡って読んでいるときに勝手に飛ばさない）。
- 追従の再開条件: アクティブ行が再び可視範囲に入る、または最新（最下部）付近へ戻したとき。
- プログラムによるスクロールと手動スクロールを区別し、誤判定を防ぐ
  （プログラムスクロール直後の短時間の `scroll` イベントを手動と誤認しない）。

### 設定

- 専用の設定トグルは設けない。履歴パネル表示中（`showTranscriptPanel`）は常時 ON。
- 理由: マーカーが控えめで邪魔にならず、設定画面をシンプルに保てるため。
  自動スクロールの ON/OFF が必要になれば後から追加できる。

## 触るファイル

- `src/content/transcriptPanel.ts` — `.active` のスタイル追加、`updateActiveByTime` と
  賢い追従スクロールの実装、行ごとの `videoTime` 保持。
- `src/content/index.ts` — 再生位置の監視（`timeupdate` / `seeked` / `append` 後）と
  パネルへの通知の配線。
- `tests/transcriptPanel.test.ts` — 現在行ハイライトのテストを追加。

## テスト方針

- パネルへ複数行を `append` した後 `updateActiveByTime` を呼び、正しい行へ `.active` が
  付くことを検証する（境界: 最初の行より前 / 巻き戻し / 履歴空 / ちょうど境界時刻）。
- 追従スクロールは jsdom の制約があるため、追従可否の判断ロジックを検証できる粒度で扱う。

## 非目標（YAGNI）

- マーカーや自動スクロールの専用設定トグルは作らない。
- 行の経過時間表示や、未来/過去の行のスタイル分け（薄く表示など）は今回行わない。
